## 项目介绍

这是一款面向 **研究生与科研场景** 的智能体系统：把「找论文、读论文、记要点、跨文献思考」串成一条闭环。用户可以用自然语言 **检索并拉取文献**（如 arXiv 等来源），并对论文 **自动阅读、抽取与总结**，减少手工下载与通读全文的时间成本。

针对 **「读完一篇忘一篇」** 的痛点，系统将阅读过程中沉淀的 **核心观点、方法要点** 等写入 **向量知识库**，支持后续 **语义检索与问答**。这样在 **同一研究方向** 下，可以把多篇文献的要点 **放在一起对照**，更容易发现 **交叉启发与创新切入点**。

同时实现 **领域 / 租户隔离**：不同课题、不同方向可以维护 **相互独立的知识库**，检索时 **不会混入无关领域的片段**，避免「搜到一堆用不上的知识」干扰判断，保证对话与调研都建立在 **当前方向的有效上下文** 之上。

## 技术栈

FastAPI、Uvicorn、SSE（`sse-starlette`）、LangChain、**LangGraph**、Milvus、可选 **Elasticsearch 混合检索**、Redis（会话历史与可选滚动摘要）、Pydantic Settings、OpenAI 兼容 API（如 OpenRouter）、`arxiv` 等。

## 个人职责（可按你实际参与情况增删）

负责 **AI Agent 架构设计**：基于 LangChain 框架设计并实现了多个 AI Agent，包括：

- **RAG 检索**：**Markdown 语义分块亮点**——先按 `# / ## / ###` **标题粗分章节**，再在章内用 **`SemanticChunker`（embedding 语义断点）** 切分，避免纯字符窗口在句中硬切、提升入库片段语义完整性；对超长块用 **tiktoken 硬顶** 二次切分，兼顾向量维度与上下文预算。`.txt` 等可走 **层级 tiktoken 父子块**（子块检索、父块展开）。**检索侧**：**Milvus** 向量检索、可选 **ES 混合召回**；**门控 / 超时降级 / 观测日志**；工具 **`retrieve_knowledge`** + **租户隔离**。
- **兜底与可降级**：RAG 侧检索超时、门控不达标时降级，避免卡死与无据强答；论文终稿流式润色失败时回退为章节拼接稿；工作流异常经错误收尾节点统一兜底并向 SSE 透出阶段信息，避免任务静默失败。
- **记忆工程**：支持 **短期记忆**（Redis 多轮对话历史、可选滚动摘要降本）与 **长期记忆**（可配置工具写入/读取），保障连续对话与跨轮上下文。
